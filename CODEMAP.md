## How-To Recipes for AI Agents

### Add a new file format renderer

1. Create `src/renderers/foo-renderer.js` with a class:
   ```js
   'use strict';
   class FooRenderer {
     render(buffer, fileName) { /* returns DOM element */ }
     analyzeForSecurity(buffer, fileName) { /* returns findings object */ }
   }
   ```
2. Add format detection in `src/app/app-load.js` ‚ÄĒ find the extension-to-renderer
   switch/if-chain and add your format.
3. Add `src/renderers/foo-renderer.js` to `JS_FILES` in `build.py` (before `app-core.js`).
4. Add viewer CSS (if needed) to the appropriate CSS file under `src/styles/`.
5. Add file extensions to `file-input accept` attribute in `build.py`.
6. Run `python build.py` and test with a sample file.
7. Run `python generate-codemap.py` to update this map.

### Add a new YARA rule

1. Choose the appropriate `.yar` file under `src/rules/` by category.
2. Add your rule following the existing pattern (rule name, meta, strings, condition).
3. Run `python build.py` to rebuild.
4. Run `python generate-codemap.py` to update this map.

### Modify sidebar content

1. Read `src/app/app-sidebar.js` ‚ÄĒ the main entry point is `_renderSidebar()`.
2. Each sidebar section has its own method (`_renderFileInfoSection`, `_renderMacrosSection`, etc.).
3. Use the symbol map above for precise line ranges.

### Modify CSS styles

### Add IOCs / security findings from a renderer

The `analyzeForSecurity(buffer, fileName)` method must return a findings object:
```js
{
  detections: [],         // Array of { name, description, severity }
  interestingStrings: [], // Array of IOC items (see below)
  riskLevel: "low",       // "low" | "medium" | "high" | "critical"
  riskScore: 0,           // Numeric ‚ÄĒ thresholds: ‚Č•10 medium, ‚Č•30 high, ‚Č•50 critical
  summary: "",            // Human-readable summary string
  formatSpecific: [],     // Array of { label, value } metadata pairs
}
```

**IOC item structure** ‚ÄĒ each item in `interestingStrings` or `externalRefs`:
```js
{
  type: IOC.URL,          // REQUIRED ‚ÄĒ must use IOC.* constants from constants.js
  url: "value",           // REQUIRED ‚ÄĒ the display value (URL, IP, email, path, etc.)
  severity: "medium",     // REQUIRED ‚ÄĒ "critical" | "high" | "medium" | "info"
  note: "SAN (DNS)",      // Optional ‚ÄĒ shown as sub-line with "‚Ü≥" prefix
}
```

**IOC type constants** (defined in `src/constants.js`):
| Constant | Value | Use for |
|----------|-------|---------|
| `IOC.URL` | `"URL"` | URLs, domains, hostnames |
| `IOC.EMAIL` | `"Email"` | Email addresses |
| `IOC.IP` | `"IP Address"` | IPv4/IPv6 addresses |
| `IOC.FILE_PATH` | `"File Path"` | File system paths |
| `IOC.UNC_PATH` | `"UNC Path"` | UNC network paths |
| `IOC.HASH` | `"Hash"` | File hashes |
| `IOC.COMMAND_LINE` | `"Command Line"` | Command line strings |
| `IOC.PROCESS` | `"Process"` | Process names |
| `IOC.HOSTNAME` | `"Hostname"` | Host/computer names |
| `IOC.USERNAME` | `"Username"` | User account names |
| `IOC.REGISTRY_KEY` | `"Registry Key"` | Windows registry keys |
| `IOC.YARA` | `"YARA Match"` | YARA rule matches (shown in Detections, not IOCs) |
| `IOC.PATTERN` | `"Pattern"` | Pattern matches (shown in Detections, not IOCs) |
| `IOC.INFO` | `"Info"` | Informational findings (shown in Detections, not IOCs) |

**‚ö† Common pitfall:** Never use bare strings like `"url"`, `"ip"`, `"domain"`, `"email"` ‚ÄĒ
always use `IOC.*` constants. The sidebar filters by exact type string to separate
IOCs from detections. Items with `IOC.YARA`, `IOC.PATTERN`, or `IOC.INFO` types are
shown in the "Detections" section; all other types appear in the "IOCs" section.
