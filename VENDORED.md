# Vendored Libraries

Loupe is **fully offline** — no npm, no lockfile, no CDN. Every third-party
JavaScript library is committed under [`vendor/`](vendor/) and inlined into
[`docs/index.html`](docs/index.html) at build time.

This file pins the **exact bytes** of each vendored library by SHA-256.
Any upgrade must rotate the corresponding hash here, so every supply-chain
change is visible in `git diff`.

Each release also ships a machine-readable **[CycloneDX 1.5](https://cyclonedx.org/)**
SBOM (`dist/loupe.cdx.json` locally, `loupe.cdx.json` on the release)
generated from this table by
[`scripts/generate_sbom.py`](scripts/generate_sbom.py) — regenerate it
locally with `python make.py sbom` or `python scripts/generate_sbom.py`.
The SBOM is the same data as the table below, shaped for supply-chain
tooling (Dependency-Track, Trivy, etc.).

## Generated vendored assets

`vendor/geoip-country-ipv4.bin` is unique among the vendored files: it is
not a third-party library you can upgrade by dropping in a newer release.
It is **regenerated** by running [`scripts/fetch_geoip.py`](scripts/fetch_geoip.py),
which downloads the five Regional Internet Registry "delegated-stats"
files (public-domain reference data published by ARIN, RIPE NCC, APNIC,
AFRINIC, LACNIC under the NRO redistribution policy), filters to IPv4
allocations, coalesces adjacent same-country runs, overlays IANA reserved
blocks, and writes a compact ~850 KB binary used at runtime to power the
Timeline view's bundled IPv4-to-country lookups.

Refresh recipe (also automated as a monthly PR via
`.github/workflows/refresh-geoip.yml`):

```bash
python scripts/fetch_geoip.py            # downloads + writes vendor/geoip-country-ipv4.bin
# Paste the printed SHA-256 into the table row above.
python make.py                            # rebuild docs/index.html with the new bytes
git add vendor/geoip-country-ipv4.bin VENDORED.md
git commit -m "chore(vendor): refresh GeoIP RIR data (YYYY-MM-DD)"
```

The bundled GeoIP data is country-only and RIR-allocation-accurate. Users
who need finer-grained or fresher data can upload their own MaxMind
`.mmdb` (or `.mmdb.gz`) file via Settings; that file lives only in the
local browser's IndexedDB and never enters the bundle or the repo.

## Verifying

Compare the live file hash against the table above:

```powershell
# Windows / PowerShell
Get-FileHash -Algorithm SHA256 vendor\*.js
```

```bash
# Linux / macOS
sha256sum vendor/*.js
```

A mismatch means a vendored file has been altered since its pinned release —
treat it as a potential supply-chain incident and investigate the diff before
building, shipping, or merging.

## Adding or upgrading a library

See [CONTRIBUTING.md § Adding or Upgrading a Vendored Library](CONTRIBUTING.md#adding-or-upgrading-a-vendored-library)
for the full recipe. The non-negotiable rule: a vendor change without a
matching row update here (version + SHA-256) is a broken commit.
